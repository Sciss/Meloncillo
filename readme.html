System Requirements

Meloncillo requires a Java 1.4 Runtime Engine. If you are using MacOS 10.3 you already have one installed. On Linux i386 and Win32 you can download the VM from Sun (java.sun.com/j2se). Note that Blackdown for Linux PPC has not yet a version above 1.2 as of September 2004. IBM might have a 1.4 for Linux PPC? Meloncillo has been successfully tested on Windows with Java 1.5.

Details of required system performance depend on the kind of realtime plug-ins used. For Apple MacIntosh computers, a G4 800 MHz with 512 MB RAM or better is recommended.

For the sound synthesis it is recommended to install SuperCollider which can be used in realtime and offline mode or to install CSound which can be used only in offline mode. SuperCollider file releases can be found at sourceforge.net/projects/supercollider. SuperCollider is currently developed for MacOS X, Linux and Windows.

If you want to use CSound with Meloncillo, please install a version 4.23f11 or higher. Don't use older versions because they are broken (e.g. 4.23f03) at least for the provided demo scripts. File releases for CSound can be found at sourceforge.net/projects/csound or www.csounds.com. Note for MacOS: the binary from Anthony Konzar 4.23f12 won't work because of java's unix line endings. Use the one from Matt Ingalls instead which comes with MacCsound (www.csounds.com/matt).

Launching

On MacOS X simply double click the Meloncillo application icon. On Linux, open a terminal, cd into the Meloncillo folder and type

> sh Meloncillo.sh

On Windows, double-click on Meloncillo.bat. If this does not work, open a terminal from the Accessories menu inside the Start menu, change into Meloncillo's folder and launch the Meloncillo.bat file again. Watch error messages in the terminal. It has been reported that on some computers the Sun Java VM doesn't put the java command in the default search path. You can test this by typing java -version in the terminal. If your computer says, java could not be found, search your harddisk for this file. It is usually placed in a folder like c:\windows\system32\java\jre1.4.2_08\bin. If you find this file, replace java in Meloncillo.bat by the full path.

If you want to use SuperCollider for realtime synthesis, before enabling realtime mode, open a terminal, cd into the "SuperCollider_f" folder and type (for eight channel output) 

> ./scsynth -o 8 -u 57110

Change History

version 0.73
april 2005
  • new session file format using explicit DTD
  • receiver tables and transmitter trajectory stored in normal AIFF files
  • new sector receiver class
  • improved realtime plug-in communication
  • introduction of groups (sub-matrices). user-image per group
  • solo and mute flags for session objects
  • changed and largely extended keyboard shortcuts which mimic common shortcuts in protools, logic pro, photoshop and peak ;-)
  • plug-ins can define session object properties, such as audio file or audio channel
  • separate tool bars for surface and timeline
  • surface tuning fork tool, timeline selection and zoom tool
  • almost complete object based online help system.
  • massive internal class structure changes (introduction of SessionObject superclass etc.)
  • user-friendly cross-platform installer using IzPack
  • improved amplitude matrix plug-in
  • experimental binaural convolution plug-in
