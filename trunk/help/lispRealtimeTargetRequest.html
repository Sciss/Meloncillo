LISP Functional Extensions : target-request (Realtime Synthesis)

(target-request <source> <params> <medium>)

this function must be called within the prepare function. it tells meloncillo that the plug-in server application will send trigger sync information. this is closely linked to supercollider's ability to send "/tr" OSC messages.

source must be "SYNC" to indicate trigger sync. params must be integer 0 at the moment. medium must be a handle to a datagram-channel which will be used to wait for incoming OSC messages. this channel must be different from the one used by the plug-in to send messages to supercollider and used for stream data sends! to make scsynth use this distinct channel for trigger messages, be sure to register it using the "/notify" command. see the plug-in case study for an example of the sync usage.
