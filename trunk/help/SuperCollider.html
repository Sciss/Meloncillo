SuperCollider

SuperCollider (SC) is a realtime sound synthesis software that is -- at the moment -- used by all of the realtime plug-ins for Meloncillo. SC was developed by James McCartney for MacOS 9 -- SC 2 -- and ported to MacOS X -- SC 3. The plug-ins use one part of SC 3, the synthesis server (scsynth). SC is now in the public domain and developed by many people for different platforms. The server is available for MacOS X, Linux and Windows at the moment. While a future version of Meloncillo will include a launcher utility, for now scsynth has to be started separately by you, if you wish to use realtime plug-ins based on SC. To do so:

  • Download and install SC. It is hosted by SourceForge, the website is sourceforge.net/projects/supercollider.
  • Start the server, an executable file called scsynth. On Windows, you'll have a separate launcher utility called ScSpawn(?) which allows you to select runtime options in a window before the server is started. On Mac OS, open a terminal window. The terminal application is located in /Applications/Utilities (this folder is localized, e.g. on a german computer it's called /Programme/Dienstprogramme). A window with a socalled prompt will be opened. Change into SC's folder, which for a typical installation is /Applications/SuperCollider_f. Then run scsynth, as depicted in the graphic below:
  • As you see, cd <path-name> will jump into the given folder. ./scsynth runs the programme, were ./ means, the terminal should look in the current folder.
  • You have to specify a few runtime options. Each option is written as a hyphen followed by an alphanumeric character, for some options followed a space character and a parameter. The communication port specified by -u <port-number> is essential. Usually you will use -u 57110. Only be sure to match this number with the setting "SuperCollider OSC Socket" in the plug-in tab of meloncillo's preferences! This port is used by meloncillo to send control data to the SC server. A nice sideeffect of this network communication is that the server can run on a machine different from the one that is used by meloncillo. For this to work, change the IP address in the preference field mentioned above. Scroll down this help document to find a checklist for running scsynth on a different computer.
  • The -o <output-number> and -i <input-number> specify the number of channels of your audio card. The input-number is only important if you use a plug-in with live input instead of soundfile input. The channel numbers needn't match the number of channels your audio card offers. Increasing the number does not make sense, but if you have an eight-channel card and you are working with a stereo-patch, you can well tell scsynth to use only two audio output channels as shown in the graphic above.
  • If your computer has more than one soundcard, you can use -H <hardware-device-name> to specify the name of the soundcard to use (you have to use a recent version of SC because this option is rather new). Use double-quotes if your card name contains space characters, e.g. -H "Built-in Audio". To find out the name of your soundcard, start scsynth without the -H option, and you will get a list of known devices such as:
Number of Devices: 2
   0 : "Built-in Audio"
   1 : "DIGI96 PAD"
  • The option -S <hardware-sample-rate> is used to explicitely set (or change) the sound card's sample rate. Not all cards may support this, e.g. on Mac OS, the built-in card cannot operate on frequencies other than 44100 Hz. The sample rate should correspond to the rate used by your sound files.
  • The other options are for fine tuning, you will rarely need them. Refer to the file "Server-Architecture.rtf" in the SC folder for an explanation of the command-line options.

Running SC on a separate machine

The realtime plug-ins communicate with SuperCollider through a protocol called OSC, acronym for Open-Sound-Control, which was developed by Matt Wright and others at the CNMAT in Berkeley. This protocol is transport-independant but in our case will use the UDP protocol. This is a common network protocol which allows SC to run on any other machine in a network. For the communication to work, you have to specify the IP address of the machine that runs SC and the port used by scsynth. This is done in the "SuperCollider OSC Socket" preferences field in Meloncillo. Open your computer's network preference to find out your machine's IP address. When running SC on the same computer as Meloncillo, use the special IP address "127.0.0.1", the socalled localhost. If the communication is not working, check your computer's firewall settings.

Different computers have different clocks, therefore they need to be synchronized in a realtime application. At the moment there is no way to sync Meloncillo and SC in terms of a "master" and "slave" clock. However, experience shows that this lack of sync is not a big deal in most applications. Meloncillo sends control data when it receives trigger messages from scsynth, so they are "in a way" synchronized. Only, the playback speed could divorce after some time. In experiments, this didn't happen even when playing back for half an hour or so without interruption, having Meloncillo on a Mac and SC on a PC, so the system clocks are quite accurate when dealing with a resolution of several milliseconds.

One issue arises with soundfile locations. The patches that come with Meloncillo assume that you are not synthesizing over a network. Therefore, the object properties of transmitters for example offer to you to browse your harddisk to select the associated soundfile. When scsynth runs on a different machine, however, it cannot access the soundfile on the Meloncillo machine, of course. To circumvent this problem, type the remote path name of the soundfile directly into the textfield! Example: You are running Meloncillo on a Mac and scsynth on a PC, connected through a local network. Your soundfiles are located on the PC, one of which has the path C:\MySounds\MySound1.WAV. When you select a transmitter in Meloncillo (on the Mac), in the Observer's Trns tab, instead of clicking the folder icon next to the "Audio File" field, click inside the textfield and type this pathname directly.
