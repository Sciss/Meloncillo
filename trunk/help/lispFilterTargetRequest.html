LISP Functional Extensions : target-request (Trajectory Filtering)

(target-request <source> <params> <medium>)

this function must be called within the prepare function. it tells meloncillo that the plug-in server application will provide a filtered version of the trajectories that should be copied back into meloncillo's session right after the plug-in returns from the render function.

source must be "TRAJ" to indicate trajectory data. params must be an integer which is the index of the transmitter. this index refers to the elements of the global collection as returned by (gethash "TRANSMITTERS" cillo). in a future version it will be possible to provide the objects themselves or their names which would make indexing more easy when processing groups. medium defines from where to receive the filtered data. it must be a pathname denoting a stereo audio-file, where channel 1 contains the X coordinates of the trajectory, and channel 2 contains the the Y coordinates of the trajectory. meloncillo can read AIFF, WAVE, some IRCAM and some AU files.
