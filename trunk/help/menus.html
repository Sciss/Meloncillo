Meloncillo menu

On Mac OS X, this menu carries the items to open the about dialog, to open the preferences pane and to quit the application. On other platforms, the about item appears in the Help menu, the preferences item appears in the Edit menu, the quit item is listed in the File menu.

File menu

  • Clear Session : removes all objects from the session, clears the timeline and resets session specific preferences. Meloncillo is a single-document-application which means there can be only one open session.
  • Open Session : opens a file chooser for a stored session to be loaded. Select the session file (".llo" extension). Meloncillo is a single-document-application which means there can be only one open session.
  • Open Recent : the submenu contains the last five(?) session files used for quick re-opening. "Clear Menu" will empty the list.
  • Save Session (As) : saves any changes made to the current session. if the session was created from scratch or if you choose "Save Session As", a file chooser pops up to select a new file name. note that a folder of this name is created and the session file itself is placed inside this folder having the same name plus ".llo" suffix. also this folder contains subfolders for receiver and transmitter specific data. Never place any other files inside a session folder because they will be erased the next time you save your session!

    There might still be an issue on Windows computers: if you see an error message saying that some files or folders could not be deleted, you will have to use "Save Session As" to store your session under another name!
  • Insert
    • New Receivers : opens a dialog to specify the number and type of receivers to insert into the session. unlike the surface pointer tool, this allows you to choose between SigmaReceivers and SectorReceivers. the receivers will be created with default locations and sizes and added to all selected groups. it's not possible at the moment to add existing receivers to a group.
    • New Transmitters : opens a dialog to specify the number and type of transmitters to insert into the session. the transmitters will be added to all selected groups. it's not possible at the moment to add existing transmitters to a group.
    • New Group from Selected Object : opens a dialog to enter the name for a newly created group. this group consists of all receivers and transmitters currently selected. it's not possible at the moment, once the group has been created, to add or remove receivers or transmitters from a group. to do so, the group must be removed and re-created.
  • Remove
    • Selected Receivers : removes the selected receivers from the session (and all groups which contain these receivers).
    • Selected Transmitters : removes the selected transmitters from the session (and all groups which contain these transmitters).
    • Selected Groups : removes the selected groups from the session. the contents of these groups, the contained receivers and transmitters, are not deleted.
  • Bounce to Disk : allows you to create autonomous audio files from your session by using an offline playback and recording plug-in. choosing this menu item will open the bounce dialog.

Edit menu

  • Undo : makes the last editing operation undone. this includes creation and deletion of objects, object property changes (e.g. renaming a receiver or moving its position), and trajectory editing. the number of undoable steps is not limited. once a session is opened, saved, or cleared, the undo buffer is erased.
  • Redo : when undo was invoked and no new editing has been performed, it is possible to redo the undone steps.
  • Cut : what this item does, depends on the window that is focussed. for the surface frame, the selected receivers are removed and put in the clipboard. for the timeline frame, the selected time span of the selected transmitters is removed and put in the clipboard.
  • Copy : what this item does, depends on the window that is focussed. for the surface frame, the selected receivers are put in the clipboard. for the timeline frame, the selected time span of the selected transmitters is put in the clipboard. for a receiver editor, the receiver (and its sensitivity tables) are put in the clipboard.
  • Paste : what this item does, depends on the window that is focussed. for the surface frame, if receivers had been put in the clipboard, they will be pasted to the surface. for the timeline frame, if trajectory portions had been put in the clipboard, they get pasted to the selected transmitters at the current timeline position. for a receiver editor, if a receiver had been put in the clipboard, it's sensitivity tables will be pasted to the receiver in the editor.
  • Delete : what this item does, depends on the window that is focussed. for the surface frame, the selected receivers are removed. for the timeline frame, the selected time span of the selected transmitters is removed. for the main window with the message log pane, the log will be cleared.
  • Select All : what this item does, depends on the window that is focussed. for the surface frame, all receivers are selected. for the timeline frame, the selected time span is set to the whole session span.

Timeline menu

  • Insert Time Span : opens a dialog to enter the length of the span in seconds. the time span is inserted at the current timeline position and affects all transmitters equally. when you have an empty session, one of the first steps is this insertion of an amount of time.
  • Selection Move Forward : the timeline selection is shifted forward such that the new selection start equals the previous selection end and the selection length is not altered.
  • Selection Move Backwards : the timeline selection is shifted backwards such that the new selection end equals the previous selection start and the selection length is not altered.
  • Filter Trajectories : opens a dialog for filtering or transforming the selected trajectories within the selection span.

View menu

  • Snap to Objects :
  • Surface Shows
    • Receiver Sensitivity : if checked, the sensitivity of the receivers are visualized by shades of grey, where white corresponds to zero sensitivity and black corresponds to a sensitivity of 1.0 or greater.
    • Equal Power Summing : if the receiver sensitivity is shown, this item affects the way how overlapping receivers are painted. if checked, an equal power summing of the individual sensitivities is used, if unchecked, the sensitivities are linearly added. For example, if two receivers overlap and their sensitivity is 0.5 each at a particular spot, linear summing would result in a total sensitivity of 1.0, hence being a black pixel, while with equal power summing the total sensitivity is sqrt( 0.5*0.5 + 0.5*0.5 ) = sqrt( 0.5 ) = 0.71, hence being a dark gray pixel. When using intensity panning for several discrete loudspeakers, equal power summing is a good graphical approximation to the actual acoustic intensity summing.
    • Transmitter Trajectories : if checked, the trajectories of the selected transmitters within the selected time span are shown as dashed green polylines. the resolution of the painted trajectory decreases when the selected timeline span increases in order to prevent too much CPU load. viewing a long span, say a few minutes, thus can show a lot of artifacts because of the polylines being too coarse.
    • User Images : if checked, the user images of all selected groups are superimposed on the surface. the user image can be specified in the group tab of the observer palette. because the images are superimposed, it is possible to use PNG images with transparency so the images are layered. due to a bug, the images are not painted often, unless you resize the surface frame. try to use small images with small file size.
    • Rulers : if checked, rulers (axis) are shown to the left and top of the surface pane, giving a hint about the underlying coordinate system.

Window menu

  • Surface (Meta+Numpad-*) : opens the surface frame and brings it to the front.
  • Timeline (Meta+Numpad-=) : opens the timeline frame and brings it to the front.
  • Transport (Meta+Numpad-1) : opens the transport palette and brings it to the front.
  • Observer (Meta+Numpad-3) : opens the observer palette and brings it to the front.
  • Meter (Meta+Numpad-4) : opens the level meter frame and brings it to the front.
  • Realtime Plug-In (Meta+Numpad-2) : opens the realtime plug-in frame and brings it to the front.

Debug menu

  • Jatha Diddler : brings up a prompt which can be used to evaluate lisp expressions. the jatha diddler help page lists all special functions available. useful also as a small calculator. for example, to know what the X and Y position of receiver in a centered circle with radius 0.25 should be for 60 degrees: 
    IN: (SETQ X (+ 0.5 (* 0.25 (COS (/ (* 60 PI) 180)))))
OUT: 0.625
IN: (SETQ Y (- 0.5 (* 0.25 (SIN (/ (* 60 PI) 180)))))
OUT: 0.28349364905389035
    you can copy and paste the results to the observer palette's receiver tab.

Help menu

  • Meloncillo Help : opens the help viewer and shows the help index.
  • Keyboard Shortcuts : opens the help viewer and shows the keyboard shortcuts.
  • Open Meloncillo Website : opens the application home page in the system's default web browser.
