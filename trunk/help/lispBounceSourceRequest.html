LISP Functional Extensions : source-request (Bounce-to-Disk)

(source-request <source> <params> <medium> <medium-options>)

this function must be called within the prepare function. it tells meloncillo to provide stream information for the plug-in. source can be either "TRAJ" or "SENSE". the stream data is available from the specified target medium, when meloncillo calls the plug-in's render function.

trajectory source

params must be an integer which is the index of the transmitter. this index refers to the elements of the global collection as returned by (gethash "TRANSMITTERS" cillo). in a future version it will be possible to provide the objects themselves or their names which would make indexing more easy when processing groups.

sensitivity source

params must be a cons cell whose car is the index of the transmitters and whose cdr is the index of the receivers. these indices refer to the elements of the global collections as returned by (gethash "TRANSMITTERS" cillo) and (gethash "RECEIVERS" cillo). in a future version it will be possible to provide the objects themselves or their names which would make indexing more easy when processing groups.

target medium

medium defines the target to which the stream data should be sent. for offline bouncing, this must be the handle of an audio-file. for trajectory data, a stereo file must be opened, where channel 1 is filled with the X coordinates of the trajectory, and channel 2 is filled with the Y coordinates of the trajectory. for sensitivity data, a mono file must be opened. it will contain the sensitivities of the receiver as scanned by the trajectory of the transmitter.

medium-options must be NIL.
