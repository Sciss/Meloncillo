Known Issues

  • Adding or removing objects, i.e. modifying the matrix, when the realtime plug-in is activated, will throw an exception.
  • Objects created after loading a lisp plug-in are not equipped with properties declared by the plug-in. To fix this, you must load the lisp source again (click Reload button).
  • The trajectory filtering dialog will not recognize changes of selected transmitters, unless closed and re-opened.
  • The bounce dialog will not recognize changes of session objects (e.g. soloing, muting), unless closed and re-opened. you have to press the close button, don't use the frame's close gadget.
  • The timeline window paint process for realtime playback (the moving red timeline position) is very slow and can seriously degrade performance on slower computers. At the moment, if this is an issue on your machine, you have to close the timeline frame when playing back with activated realtime plug-ins.
  • Switching groups can cause graphic artifacts in the timeline frame. The shading of the selected span might be invisible or have the wrong height. In this case, change the timeline frame's size or close and re-open it.
  • The user image associated with a group is often not painted, unless you change the surface frame's size.
  • Session files created with Meloncillo 0.68 cannot be opened because the file format changed. Contact me if you have old sessions that need to be converted (it can be done manually but requires some work).
  • The java docs of the source code seriously need an update.
  • On Mac OS X, there are two versions of CSound, one by Anthony Kozar and one by Matt Ingalls; the former uses wrong line endings for text files and cannot handle the newline character created by the Meloncillo plug-ins. You can use the CSound binary that is included in MacCsound.
  • The noise generator cannot work with beta > 0.0 when used in trajectory filtering because it produces jumps between processing blocks. As a workaround for now use a beta of 0.0 (white noise) and apply smoothing afterwards.
  • The realtime plug-in frame should be left open when using realtime synthesis.
  • Timeline zoom tool rubberband has wrong display position if tracklist is vertically scrolled.
  • Having different plug-in selection in bounce and realtime dialog results in confusing property fields of the observer (you cannot see which fields belong to which plug-in).
  • On non-Mac OS X systems, the windows that do not have a menu bar do not react to keyboard shortcuts
  • ... (a lot more)

Future Plans

  • Finish manual pages (--sigh--)
  • The Lisp plug-ins should be able to react instantaneously to object property changes by implementing a kind of call-back function.
  • The Jatha Diddler should be more ergonomic with a multiline text input and optional startup file for user-defined functions.
  • A small integrated SuperCollider launcher utility.
  • Blending mode should allow for polar interpolation.
  • Saving the session should omit superfluous processes, e.g. not save transmitters again if they haven't been modified.
  • There should be a way for plug-ins to define default values for object properties, e.g. for new receivers a way to assign reasonable audio bus values. Also the layout of the properties should be controllable, or at least be alphabetical.
  • Storing and recalling settings of the plug-in dialogs.
  • Adding a scratch-board to place common transmitter movements, a kind of visible clipboard or library that can hold various objects.
  • The geometric surface tools should immediately influence realtime transmitter movements even before being rendered. There should be a graphical hint for the direction of the movement.
  • A matrix window should show the session configuration and allow the adding or removal of objects to existing groups.
  • The object tables on the surface frame should not use JTable, so that modifier keys will be available for switching to solo-safe mode for example.
  • A surface tool for affine transforms (rotation for example) of trajectories.
  • A way to skip signal delays in trajectory filtering, for example when using long filters as in the csound smooth example for low frequencies.
  • Storing and recalling a set of window arrangements.
  • Get the Meloncillo-Max/MSP Bridge running. Try to bridge with Pure Data.
  • Spatial and time based markers.
  • Create an OSC server interface, allow the plug-ins to control the application to a certain extent. One can imagine a plug-in that will arrange all receivers in a certain way for example.
  • Working line tool for vector editors. Parameters for smoothing process.
  • Importing objects from other sessions
  • ... (a lot more)
