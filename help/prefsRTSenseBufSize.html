Plug-Ins Prefs : Realtime Sense Buffer Size

This is the preferred buffer size in milliseconds for sense data in realtime synthesis mode. Its value is available in LISP scripts via (gethash "SENSEBUFSIZE" (gethash "PREFERENCES" cillo)). Depending on the sense rate, there is a physical limit determined by the maximum UDP packet size of approx. 8 KB. Since sense rate is sent as 32bit floating point data, this means that no more than c. 1500 frames should be sent at once. Hence, since data is sent as a half-size buffer, for a sense rate of 1 kHz, the sense buffer size should be no greater than three seconds (or six seconds at 500 Hz). Also the greater the sense buffer size, the more CPU intensive is each data transfer. However, the smaller the sense buffer size, the faster the transmission must be. If you recognize drop-outs in realtime mode, either increase the sense buffer size or decrease the sense buffer rate.
