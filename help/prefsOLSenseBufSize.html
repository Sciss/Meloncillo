Plug-Ins Prefs : Offline Sense Buffer Size

This is the preferred buffer size in milliseconds for sense data in offline (bounce-to-disk) mode. Its value is available in LISP scripts via (gethash "SENSEBUFSIZE" (gethash "PREFERENCES" cillo)).

Warning! As of the time of this writing, there is a severe bug in SuperCollider's nonrealtime mode which is used for bouncing: Whenever the rate of messages is to small, the DiskIn UGen will produce strange artifacts like looping the soundfiles etc. If you encounter this, use a smaller offline sense buffer size which results in increased buffer update rate and usually removes the described problem, though the rendering will be substantially slower, unfortunately.
