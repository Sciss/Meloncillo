LISP Functional Extensions : Audio File I/O commands

(audio-file-open <id> <path> [<type> <res> <rate> <channels>])
(file-close <id>)

audio-file-open can be used with two or six arguments. the two arguments call will open the audio file denoted by the path for reading and assign a handle to the lisp symbol id, as explained in the oboundp document. this handle can then be used to close the file. audio-files are used to exchange stream data in trajectory filtering and bouncing. see the source-request and the target-request documents. there are no functions to read or write directly from within a lisp script.

when opening an audio file for writing, type can be one of the strings "AIFF", "WAVE", "IRCAM", "AU" or "RAW". res can be one of the strings "int16", "int24", "int32" or "float32". rate is an integer or real that specifies the sampling rate. channels is the number of audio channels (1 for sense data, 2 for trajectory data).
