Writing LISP Plug-Ins : Functional Extensions

a few addtional lisp functions are provided by meloncillo:

  • current-time-millis : returns current system time in milliseconds.
  • byte-buffer-alloc : allocates a byte buffer.
  • byte-buffer-write : writes the data into a byte buffer.
  • byte-buffer-free : frees the memory occupied by a byte buffer.
  • datagram-channel-open : opens a (UDP) datagram channel for OSC communication.
  • datagram-channel-write : writes contents of a bytebuffer to a datagram channel
  • datagram-channel-close : closes and frees a datagram channel.
  • file-open : opens a file for reading or writing.
  • file-write : writes the contents of a bytebuffer to a file.
  • file-close : closes a file.
  • audio-file-open : opens an audio file for reading or writing.
  • file-delete : deletes a file.
  • temp-file-make : creates a new empty temporary file.
  • format-apply : formats a pattern string with variable substitution.
  • format-parse : extracts variables out of a formatted string following a pattern.
  • log : calculates the logarithm of number
  • pow : raises a number to the power of another number
  • logand : calculates the bitwise AND of two numbers.
  • logior : calculates the bitwise OR of two numbers.
  • logxor : calculates the bitwise XOR of two numbers.
  • oboundp : the predicate is T if a java object with a given identifier has been created.
  • osc-bundle-send : sends OSC commands to a network socket or file.
  • osc-bundle-send-and-wait : sends OSC commands to a network socket or file, and waits for a reply.
  • path-concat : concatenates a parent directory and a file to a new path name.
  • path-split : extracts a parent dir and a file from a path name.
  • println : prints an object to the console and adds a newline character.
  • regex-match : matches a string against a regular expression and returns the group boundaries
  • substring : returns a portion of a given string.

further more, a constant PI is defined (3.1415926...).

plug-in only functions

the functions above can also be used in the jatha diddler. in plug-in scripts, a few more functions are available:

  • gadget-make : when used inside a special function block create-gui, this function defines a new graphical user interface element in the plug-in's frame.
  • session-property-add : when used inside a special function block create-gui, this function defines a new property of session objects (transmitters and receivers) which is displayed in the observer palette.
  • execute : synchronously executes another application or shell script.
  • progression-set : updates the main window's progress bar position.

    • a special symbol cillo (a hashtable) is passed to special functions defined by the plug-in source code. see the plug-in concept document.

    plug-in host specific functions

    special functions exist to communicate stream data (sensitivities or trajectories) between meloncillo and a server application. there exact behaviour depends on the kind of plug-in host. for the filter trajectories plug-in:

    • source-request : asks meloncillo to provide trajectory input data.
    • target-request : tells meloncillo about processed trajectories that should be re-inserted into the session.

    for the realtime synthesis plug-in:

    • source-request : asks meloncillo to send sensitivity or trajectory input data to an OSC port.
    • target-request : tells meloncillo about incoming synchronization messages.

    for the bounce-to-disk plug-in:

    • source-request : asks meloncillo to write sensitivity or trajectory input data to an harddisk file.
