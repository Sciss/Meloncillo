Plug-Ins Prefs : Audio Hardware Inputs

Meloncillo does not care about audio processing at all. The value specified here is available to LISP scripts via (gethash "INPUTCHANNELS" (gethash "PREFERENCES" cillo)). For example, in SuperCollider, audio busses are automatically reading from the sound card's input channels for indices smaller than the number of specified input channels.

Note that this field doesn't influence realtime SuperCollider which is started manually. However, for the plug-ins to not get confused, this value should be greater or equal than the number of audio inputs specified when launching scsynth!
