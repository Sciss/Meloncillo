LISP Functional Extensions : Datagram Channel commands

(datagram-channel-open <id> <host> [<port>])
(datagram-channel-write <id> <object>)
(datagram-channel-close <id>)

a datagram channel is a network channel over which messages, typically open sound control (OSC) messages, are sent from one application to another. to communicate with supercollider for example, one has to send OSC messages through such a channel. while sc supports two protocols, UDP and TCP, these lisp functions allow the use of the user datagram protocol (UPD) only.

instead of specifying the target address in each write command, the target socket is specified once in the creation command of the channel. the host is a string argument which can be either an IP number such as "12.110.110.204" or a host name such as "localhost" or "www.nsa.gov" (a really cool organization of the world's largest democracy). an IP address identifies the target computer, where "localhost" is an abbrevation for the same computer as the message sender. the port is an arbitrary number, typically something between 1024 and 65535. the target application publishes its communication port which must be known by the sender. when omitting the port argument, a default of 57110 is used, which is the traditional port of supercollider.

when opening a channel, this channel is assigned to a lisp symbol id which serves as handle for writing and closing the channel. see the oboundp for more information about object binding.

when writing to the channel, the parameter object has to be a previously filled byte buffer. usually, you will want to use the simpler OSC commands.
